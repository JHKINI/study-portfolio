# 자료구조 기초 정리

## 1. 자료구조란?

자료구조는 데이터를 효율적으로 저장하고 관리하는 방법이다.

### 선형 자료구조

데이터가 순서대로 이어져 있는 구조이다.

예시:

- 리스트
- 큐
- 덱
- 스택

### 비선형 자료구조

데이터가 일렬로 이어져 있지 않고, 여러 방향으로 연결되는 구조이다.

예시:

- 트리
- 그래프
- 집합

---

## 2. 알고리즘이란?

알고리즘은 문제를 해결하기 위한 절차나 방법이다.

좋은 알고리즘은 다음 조건을 만족해야 한다.

| 조건 | 의미 |
|---|---|
| 명백성 | 각 단계가 모호하지 않아야 한다 |
| 유한성 | 언젠가는 반드시 끝나야 한다 |
| 유효성 | 실제 코드로 구현할 수 있어야 한다 |

---

## 3. 알고리즘 성능

알고리즘의 성능은 보통 다음 기준으로 평가한다.

- 계산 속도
- 메모리 사용량
- 기능 달성도
- 실제 구현 가능성

---

## 4. 시간 복잡도와 빅오 표기법

알고리즘의 효율성을 비교할 때 연산 횟수를 기준으로 시간 복잡도를 분석한다.

### 빅오 표기법

| 표기 | 의미 |
|---|---|
| Big-O | 상한. 최악의 경우 어느 정도까지 느려질 수 있는지 |
| Big-Ω | 하한. 아무리 빨라도 이 정도 연산은 필요하다는 의미 |
| Big-Θ | 상한과 하한이 같은 경우. 평균적인 정확한 성장 정도 |

시간 복잡도는 내가 만든 알고리즘이나 자료구조가 기존 방식보다 얼마나 효율적인지 비교하는 기준으로 사용할 수 있다.

---

## 5. 리스트

리스트는 데이터를 순서대로 저장하는 자료구조이다.

### 리스트의 추상 자료형 ADT

리스트는 내부 구현 방식과 상관없이 다음과 같은 기능을 제공할 수 있다.

- 삽입
- 삭제
- 탐색
- 수정
- 출력

### 배열 기반 리스트

배열 기반 리스트는 메모리 상에서 연속된 공간에 데이터를 저장한다.

장점:

- 인덱스를 이용한 접근이 빠르다.

단점:

- 용량 변경이 어렵다.
- 중간 삽입과 삭제가 비효율적이다.

---

## 6. 스택

스택은 나중에 들어온 데이터가 먼저 나가는 구조이다.

이를 LIFO 구조라고 한다.

> Last In First Out

### 주요 개념

| 개념 | 의미 |
|---|---|
| Push | 데이터 삽입 |
| Pop | 데이터 삭제 |
| Top | 가장 위에 있는 데이터 |
| Overflow | 저장 공간이 가득 찬 상태에서 추가하려는 경우 |
| Underflow | 데이터가 없는데 삭제하려는 경우 |

### 활용 예시

- 괄호 검사
- 되돌리기 기능
- 미로 탐색 DFS
- 함수 호출 관리

---

## 7. 큐

큐는 먼저 들어온 데이터가 먼저 나가는 구조이다.

이를 FIFO 구조라고 한다.

> First In First Out

### 주요 개념

| 개념 | 의미 |
|---|---|
| Enqueue | 데이터 삽입 |
| Dequeue | 데이터 삭제 |
| Front | 가장 앞 데이터 |
| Rear | 가장 뒤 데이터 |

### 활용 예시

- 운영체제 작업 처리
- 프린터 대기열
- 버퍼
- BFS 탐색

---

## 8. 우선순위 큐와 미로 탐색 아이디어

일반 큐는 먼저 들어온 데이터를 먼저 꺼내지만, 우선순위 큐는 우선순위가 높은 데이터를 먼저 꺼낸다.

미로 탐색에서는 현재 위치에서 갈 수 있는 칸들을 큐에 넣고, 출구와 가까운 칸부터 꺼내도록 만들 수 있다.

예를 들어 출구까지의 거리를 `d`라고 하면,

- `d`가 작을수록 출구와 가까움
- 가까운 칸을 먼저 꺼내기 위해 `-d` 또는 우선순위 값을 조정할 수 있음

이 방식은 어린이용 미로 탈출 게임이나 보물상자 아이템 찾기 같은 지능 훈련 앱에 활용할 수 있다.

---

## 9. 덱

덱은 양쪽 끝에서 삽입과 삭제가 모두 가능한 자료구조이다.

> Double Ended Queue

### 특징

- 앞에서 삽입 가능
- 앞에서 삭제 가능
- 뒤에서 삽입 가능
- 뒤에서 삭제 가능

### 활용 예시

- 앞뒤 이동이 필요한 문제
- 슬라이딩 윈도우
- 양방향 대기열

---

## 10. 집합

집합은 중복되지 않는 데이터를 저장하는 자료구조이다.

### 특징

- 중복을 허용하지 않는다.
- 순서가 중요하지 않을 수 있다.
- 특정 값이 있는지 빠르게 확인할 수 있다.

### 활용 예시

- 중복 제거
- 방문 여부 확인
- 공통 원소 찾기

---

## 11. 연결 구조와 배열 구조

자료구조는 크게 배열 기반 구조와 연결 기반 구조로 구현할 수 있다.

### 배열 구조

메모리 상에서 연속된 공간에 데이터를 저장한다.

장점:

- 인덱스 접근이 빠르다.

단점:

- 크기 변경이 어렵다.
- 중간 삽입과 삭제가 느릴 수 있다.

### 연결 구조

데이터들이 포인터나 참조로 연결되어 있는 구조이다.

장점:

- 삽입과 삭제가 비교적 자유롭다.
- 크기 변경이 유연하다.

단점:

- 특정 위치에 바로 접근하기 어렵다.

---

## 오늘 배운 내용 요약

오늘은 자료구조와 알고리즘의 기본 개념을 정리했다.

자료구조는 데이터를 저장하는 방식이고, 알고리즘은 문제를 해결하는 절차이다.  
리스트, 스택, 큐, 덱, 집합 같은 자료구조는 각각 다른 특징을 가지고 있으며, 문제 상황에 따라 적절히 선택해야 한다.

특히 스택은 DFS, 큐는 BFS, 우선순위 큐는 거리나 중요도에 따라 먼저 처리할 대상을 정할 때 사용할 수 있다.
